Локальный CDN & Кэширование ассетов

Устройство двухэтапного кэширования и оптимизация трафика при загрузке ассетов.

Пайплайн загрузки ассетов (Локальный CDN)

Для минимизации сетевого трафика и быстрого запуска игры Fodinae использует интеллектуальную систему кэширования по протоколу TCP с валидацией изменений через хэш-суммы ETag.

Алгоритм загрузки (GetAssetAsync):

  1. RAM-кэш (AssetCache): При запросе ассета сначала проверяется его наличие в оперативной памяти. Если ассет найден, он возвращается мгновенно.
  2. Дисковый кэш (PersistentAssetCache):
    • Если в RAM ассета нет, проверяется его наличие на диске (в папке Application.persistentDataPath).
    • Если файл на диске отсутствует, отправляется запрос на сервер.
    • Если файл есть на диске, клиент считывает локальный ETag и отправляет его серверу в пакете запроса для валидации.
  3. Валидация по TCP:
    • Клиент упаковывает запрос в сетевой пакет RuntimeAssetRequestPacket, содержащий имя файла и локальный ETag.
    • Если файл на сервере не менялся, сервер возвращает пакет RuntimeAssetPacket с пустым буфером данных (длина 0) и ETag. Клиент считывает файл с диска в RAM и возвращает его.
    • Если файл обновился или отсутствует в кэше, сервер присылает новые байты в пакете RuntimeAssetPacket. Клиент сохраняет новые байты на диск, вычисляет локальный ETag и возвращает ресурс.

ClientAssetLoader & Объединение запросов

Синглтон ClientAssetLoader управляет асинхронными сетевыми операциями загрузки текстур, звуков и текстовых данных.

Коалесценция запросов (Coalescing):

Когда на экране появляется много одинаковых объектов (например, кристаллы одного типа), они одновременно запрашивают одну и ту же текстуру с сервера. Чтобы не создавать десятки параллельных сетевых запросов к одному файлу, ClientAssetLoader объединяет их:

  • Если файл уже скачивается, новый запрос не запускает сетевую операцию, а регистрирует объект TaskCompletionSource.
  • Когда сетевой поток первого запроса завершается и сохраняет файл, все ожидающие асинхронные задачи завершаются одновременно, используя один и тот же результат.

Стойкий дисковый кэш (PersistentAssetCache)

Класс PersistentAssetCache отвечает за физическую запись файлов в кэш и хранение метаданных.

  • ETag-валидация: Локальные ETag рассчитываются на основе MD5 хэш-суммы данных файла с помощью утилиты ETagCalculator.
  • Строка ETag сохраняется в ассоциативный файл-спутник с расширением .etag (например, image.png.etag) прямо в папке кэша рядом с оригинальным файлом, что позволяет кэшу быть стойким между перезапусками игры.

Динамические изображения (DynamicImage)

Компонент DynamicImage расширяет стандартный компонент Unity UnityEngine.UI.Image. Он позволяет задавать источник спрайта в виде URL-ссылки на сервер CDN. В методе инициализации компонент отправляет асинхронный запрос в ClientAssetLoader и автоматически обновляет картинку на экране по мере загрузки.
