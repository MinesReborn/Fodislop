Аудиоподсистема Fodinae

Полное архитектурное руководство по устройству звука, FMOD Studio бэкенду, иерархии шин и пространственной акустике.

Обзор Архитектуры

Аудиоподсистема Fodinae спроектирована как независимый, высокооптимизированный звуковой домен. Он разделяет понятия семантических аудиособытий (то, что запрашивает геймплейный код) и бэкенда воспроизведения (FMOD Studio Engine или Unity AudioSource).

Ключевые принципы архитектуры:

  • Семантический доступ: Код игры не работает напрямую с именами звуковых файлов. Вместо этого вызываются именованные события (AudioSystem.Play("dig_rock"), AudioSystem.PlayAt("explosion", pos)).
  • Нативный FMOD Бэкенд: Управление громкостью, дакингом, Voice Limit и фильтрами происходит в FMOD Studio Engine.
  • Динамическое скачивание банков: В режиме MMO файлы FMOD-банков (.bank) скачиваются с игрового CDN через ClientAssetLoader с ETag-кэшированием и подгружаются в память FMOD в рантайме.

Точка входа: AudioSystem

Класс AudioSystem является синглтоном, живущим в DontDestroyOnLoad. Он предоставляет глобальный C# API для проигрывания звуков и управления шинами.

// Проигрывание звука интерфейса (2D)
AudioSystem.Play2D("ui_click");

// Проигрывание позиционного звука копки в мировых координатах
AudioSystem.PlayAt("dig_rock", transform.position);

// Управление громкостью шины звуковых эффектов
AudioSystem.Instance.GetBus(AudioBusType.Sfx).Volume = 0.8f;

FMOD Studio Бэкенд (FmodAudioBackend)

Класс FmodAudioBackend отвечает за взаимодействие с FMOD C++ Engine:

  1. Загрузка банков: Метод LoadAllBanks() опрашивает ClientAssetLoader и загружает банки Master, SFX, Music, Ambience, Voice через loadBankMemory. При отсутствии связи используется локальный фоллбек из StreamingAssets/Audio/.
  2. Создание голосов (CreateVoice): Преобразует событие в путь FMOD (event:/{Name}). Вызов обернут в защищенный try-catch блок: если событие отсутствует в банках, система логирует предупреждение без вызова исключений и падения сетевого сервиса.
  3. Инверсия координат Y: Автоматически транслирует 2D-координаты мира Unity в 3D-векторы атрибутов FMOD (FMOD.ATTRIBUTES_3D).

Иерархия Шин (AudioBus)

Все звуки игры проходят через строго определенную иерархию шин (AudioBus):

  • Master — Главный выходной канал проекта.
  • Sfx — Звуковые эффекты мира (копка, взрывы, шаги).
  • Music — Фоновая музыка и саундтрек.
  • Voice — Речь и всплески персонажей / диктора.
  • Ambience — Эмбиент окружения (пещеры, ветер, эхо).
  • Ui — Звуки кликов, кнопок и окон интерфейса.
  • Narrative — Сюжетные аудиозаписи и диалоги.

Пространственный Звук & Акустические Зоны

За объемное звучание и адаптацию атмосферы отвечают два компонента:

  • AudioSpatial — навешивается на GameObject (роботы, предметы) и непрерывно обновляет позицию звучания активного голоса в FMOD каждый кадр.
  • AudioZone — триггерная зона. При входе игрока в зону (например, подземелье или подводная локация) плавно меняет параметры громкости и среза частот на целевой шине.

Низкоуровневый парсинг WAV (WavUtility)

Для прямого воспроизведения сырых сэмплов без сборки FMOD-банков (например, при локальной отладке) используется WavUtility. Он разбирает заголовки RIFF/WAVE напрямую из потока байт в памяти и декодирует 16-битные PCM-данные в массив float[] без лишних аллокаций памяти.
