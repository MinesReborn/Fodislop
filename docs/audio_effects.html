Звук и Эффекты

Устройство рантайм-воспроизведения аудио с сервера и система визуальных эффектов.

AudioManager

Класс AudioManager — центральный синглтон, управляющий звуковым окружением игрока. Он содержит ссылки на основные компоненты AudioSource для зацикленного воспроизведения фоновой музыки (BGM) и контролирует громкость звуков на основе настроек, сохраняемых в PlayerPrefs.

Низкоуровневый парсинг звука (WavUtility)

Так как звуковые файлы игровых тайлов и действий (копка, удары, шаги) подгружаются динамически с сервера CDN, игра не может использовать стандартный импорт аудиофайлов в редакторе Unity. Сырые байты WAV-файлов считываются и парсятся прямо в оперативной памяти.

Этот процесс выполняет высокооптимизированный класс WavUtility:

  1. Метод ToAudioClip принимает поток байт аудиофайла.
  2. Парсер считывает стандартный заголовок формата RIFF/WAVE, находя чанки описания формата ("fmt ") и данных ("data").
  3. Извлекаются метаданные: частота дискретизации (sample rate), количество каналов и битрейт (например, 16-бит).
  4. PCM-данные конвертируются в массив значений с плавающей запятой float[] (в диапазоне от -1.0 до 1.0 путем деления исходных 16-битных значений на 32768f) без лишних аллокаций памяти.
  5. Создается новый пустой AudioClip в оперативной памяти с помощью метода AudioClip.Create, который наполняется полученными сэмплами через clip.SetData.

Пул звуков и эффектов: SFXPool

Поскольку во время геймплея одновременно может происходить множество звуковых эффектов (копка, сражения нескольких игроков), постоянное создание и уничтожение объектов AudioSource привело бы к сильной фрагментации памяти и просадкам FPS из-за сборщика мусора (GC).

Для решения этой проблемы используется класс SFXPool, реализующий паттерн Объектного пула (Object Pooling):

  • При инициализации сцены создается фиксированное количество объектов с компонентами AudioSource и визуальными контейнерами.
  • Когда коду требуется проиграть звук, он запрашивает свободный слот из пула.
  • Звук воспроизводится методом PlayOneShot.
  • По окончании воспроизведения объект автоматически освобождается и возвращается обратно в пул для повторного использования.

Пайплайн воспроизведения эффектов

Визуальные и звуковые эффекты синхронизируются на основе сетевых данных от сервера:

  1. Сервер присылает сетевой пакет SFXPacket, содержащий ID эффекта, его мировые координаты и тип.
  2. Синглтон SFXEffectManager перехватывает пакет и спавнит экземпляр эффекта.
  3. Каждый эффект представляет собой экземпляр SFXEffectInstance, который:
    • Запрашивает аудио и визуальные ресурсы через загрузчик ассетов.
    • Позиционируется в мировых координатах Unity (с учетом инверсии Y).
    • Воспроизводит звук через пулированный AudioSource из SFXPool.
    • Декодирует и проигрывает кадры анимации (GIF/PNG) или запускает Effekseer-эффект.
    • Автоматически возвращается в пул после завершения анимации и звука.

Интеграция Effekseer

Для отображения сложных 3D/2D спецэффектов (например, взрывы, заклинания, частицы) в проект интегрирована рантайм-система Effekseer с помощью компонента RuntimeEffekseerLoader. Он инициализирует систему частиц, загружает файлы эффектов во время выполнения игры и управляет их жизненным циклом.
