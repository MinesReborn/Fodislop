Сетевое взаимодействие

Архитектура асинхронного сетевого слоя и динамического построения интерфейса.

NetworkService

Класс NetworkService является центральным синглтоном диспетчеризации сетевых пакетов. Он реализует событийно-ориентированный паттерн Pub/Sub (Издатель-Подписчик) для обработки десериализованных данных.

Интерфейс подписки:

Любой менеджер или игровой скрипт может подписаться на определенный тип пакета, приходящего от сервера, не зная деталей сетевого сокета:

// Подписка на обновление координат роботов
NetworkService.Instance.Subscribe<RobotMovePacket>(OnRobotMoved);

// Отписка при уничтожении объекта
NetworkService.Instance.Unsubscribe<RobotMovePacket>(OnRobotMoved);

PacketHandler

Класс PacketHandler считывает входящие потоки байт, восстанавливает типы пакетов из внешней библиотеки darkar25.fodinae.networking и распределяет их по менеджерам:

  • WorldInitPacket, MapRegionPacket → передаются в MapManager для инициализации карты.
  • RobotSpawnPacket, RobotMovePacket, RobotDespawnPacket → передаются в RobotManager для обновления роботов на сцене.
  • PackSpawnPacket, PackDespawnPacket → передаются в PackManager для управления лежащими на земле вещами.
  • SFXPacket → передается в SFXEffectManager для спавна аудиовизуальных эффектов.

ConnectionManager & TCP Lifecycle

Управление состоянием соединения с сервером инкапсулировано в синглтоне ConnectionManager.

Жизненный цикл сокета:

  1. При старте или по запросу пользователя вызывается метод подключения к IP и порту сервера. Под капотом используется враппер над NetCoreServer.
  2. После успешного установления TCP-сессии отправляется пакет авторизации LoginRequestPacket, содержащий имя пользователя и токен.
  3. При потере соединения менеджер автоматически запускает таймер восстановления сессии (Reconnecting) с экспоненциальной задержкой.
  4. Офлайн-режим игры поддерживается через заглушку DummyConnection, имитирующую ответы сервера на запросы.

TextureStorageManager

Для оптимизации трафика при передаче пользовательских данных (аватары, текстуры кланов, кастомные скины) используется менеджер TextureStorageManager. Он запрашивает файлы через сетевые пакеты по мере необходимости, кэширует их на жестком диске и предотвращает повторные запросы.

Динамический интерфейс (Packet UI)

Уникальной особенностью Fodinae является возможность сервера динамически конструировать пользовательский интерфейс клиента без пересборки игры. Когда сервер присылает OpenWindowPacket, клиент строит окно в рантайме.

Пайплайн сборки интерфейса:

  1. Пакет OpenWindowPacket содержит древовидное описание UI-элементов, их стилей, размеров и привязок событий.
  2. Класс PacketUIBuilder делегирует сборку фабрике PacketUIBuilderFactory.
  3. Фабрика сопоставляет ID компонентов с конкретными билдерами на базе UI Toolkit:
    • CanvasPacketBuilder / PanelPacketBuilder — базовые контейнеры.
    • GridPacketBuilder — сеточные элементы.
    • TextPacketBuilder / TextBoxPacketBuilder — вывод текста и текстовые поля.
    • ButtonPacketBuilder — кликабельные кнопки.
    • SliderPacketBuilder — ползунки.
    • IntDropdownPacketBuilder / StringDropdownPacketBuilder — выпадающие списки.
    • ScrollViewerPacketBuilder — прокручиваемые области.
    • LinePacketBuilder — разделительные линии (с использованием кастомного класса UILine).
    • DockPanelPacketBuilder — докинг-панели на базе перечисления Dock (Left, Top, Right, Bottom).
