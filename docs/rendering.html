Архитектура рендеринга

Как Fodinae отрисовывает тайловый мир в один draw call.

Обзор

SingleMeshTerrainRenderer строит один Mesh из всех видимых тайлов каждый кадр. Камера определяет границы видимости, рендерер запрашивает данные из MapStorage и собирает геометрию.

UV-каналы

КаналНазначение
UV0Координаты в атласе текстур
UV1Sub-atlas rect (позиция фрейма)
UV2Размер тайла в UV-пространстве
UV3Мировые координаты + тайлинг-дескриптор
UV4Анимация: тип, скорость, оффсет
UV5Тени и рельеф
UV6Локальные UV для шейдерных эффектов

Инверсия Y

Сервер: Y растёт вниз (0 = верхняя строка).
Unity: Y растёт вверх. 
int serverY = MapManager.Instance.WorldHeight - 1 - unityY;

Поток данных

  1. PacketHandler получает MapRegionPacket
  2. Данные записываются в MapStorage.cellLayer (через WorldLayer<T>)
  3. LateUpdate рендерера проверяет MapStorage.IsReady
  4. Строится меш из видимых тайлов + фон через ComputeBackgroundMap
  5. Меш загружается на GPU с IndexFormat.UInt32

Сортировка

Террейн рисуется на Sorting Order = -1000, чтобы находиться под спрайтами роботов и UI-элементами.
