Мир и Архитектура рендеринга

Технические детали высокопроизводительного тайлового рендеринга в Fodinae.

В отличие от стандартного подхода Unity, где каждый тайл сетки отрисовывается в виде отдельного игрового объекта SpriteRenderer (что приводило бы к тысячам вызовов отрисовки и высокой нагрузке на процессор), Fodinae использует кастомный тайловый рендерер.

SingleMeshTerrainRenderer

Класс SingleMeshTerrainRenderer динамически строит и обновляет ровно один полигональный меш (Mesh) для всей видимой области террейна каждый кадр. В результате вся сетка мира рендерится на GPU за один Draw Call.

Принцип работы:

  1. В методе LateUpdate рендерер проверяет флаг готовности мира MapStorage.IsReady.
  2. На основе положения и размера ортографической камеры вычисляется ограничивающий прямоугольник (видимая область чанков 32x32).
  3. Из MapStorage считываются ID типов ячеек видимого диапазона.
  4. Рендерер генерирует вершины (Vertices), индексы треугольников (Triangles) и заполняет 7 UV-каналов, кодирующих параметры отображения тайлов.
  5. Для отрисовки меша используется формат индексов IndexFormat.UInt32, что позволяет объединять сетки размером более 65,000 вершин.
  6. Меш позиционируется на слое сортировки Sorting Order = -1000, чтобы находиться под всеми динамическими объектами (роботами, предметами) и интерфейсом.

7 UV-каналов террейна

Шейдер террейна считывает детальную информацию о тайлах из различных каналов вершинных атрибутов меша (UV0 - UV6):

Канал Тип данных Назначение / Описание данных
UV0 Vector2 Базовые текстурные координаты тайла в глобальном скомпонованном атласе текстур.
UV1 Vector4 Sub-atlas Rect (subAtlasRects). Ограничивающий прямоугольник тайла внутри текстурного атласа. Предотвращает выход за границы текстуры (texture bleeding) при анимациях и рендеринге.
UV2 Vector4 Параметры размера тайла (tileSizeUVs). x, y — нормализованный размер тайла в текстурном пространстве атласа; z — количество кадров анимации; w — высота фрейма в тайлах (для многострочных текстур).
UV3 Vector4 Мировые координаты & Битмаска (worldPositions). x, y — мировые координаты ячейки; z — битовая маска авто-тайлинга ячейки (биты 0-4); w — запакованные метаданные / ID слоя.
UV4 Vector4 Параметры анимации (animationData). x — тип анимации (loop, trigger и др.); y — скорость анимации; z — временное смещение кадра (animOffset) для рассинхронизации; w — зарезервировано (0).
UV5 Vector4 Тени и рельеф (packedReliefShadowLocalUV). x — тип текстуры (1.0 = рельефный переход, 0.0 = базовая земля); y — маска рельефа ячейки или значение затенения вершины (shadow shade); zw — координаты локального смещения вершин для проецирования теней.
UV6 Vector4 Параметры свечения и авто-тайлинга (glowData). xy — вектор направления/координат свечения магмы; z — флаги свечения ячеек и скругления краев; w — маска совпадения категорий соседних ячеек (sameCatMask).

Кастомные слои: SurfaceRenderer

Некоторые визуальные элементы тайлов требуют особого расположения по глубине или перекрытия. Для этого используется вспомогательный рендерер SurfaceRenderer, создающий два дополнительных меша:

  • Transit (Переходы): Отрисовывает меш плавных переходов градиентов между различными слоями высот террейна. Рисуется поверх базовой земли.
  • Perspective (Перспективные блоки): Отрисовывает трехмерные стены и вертикальные выступы блоков, создающие эффект глубины. Эти элементы сортируются динамически относительно роботов, используя отдельные материалы и повышенный Sorting Order.

Динамический атлас и кэширование текстур

Поскольку типы ячеек и их текстуры могут загружаться динамически с сервера CDN, игра не использует жестко прописанные ассеты в папках Resources или Addressables.

Схема загрузки текстур:

  1. WorldTextureManager считывает файлы спрайтов ячеек напрямую из файловой системы по путям, предоставленным CDN.
  2. Все загруженные текстуры упаковываются в единый динамический текстурный атлас с помощью класса TextureAtlas. Атлас возвращает структуру AtlasCoordinate для каждого типа ячейки.
  3. Для ускорения загрузки и предотвращения повторного чтения файлов используется потокобезопасный кэш CellTextureCache на базе ConcurrentDictionary<CellType, Texture2D>.

Декодирование анимаций (GIF / WebP)

Анимированные тайлы (например, лава, вода, кристаллы) загружаются из файлов GIF или WebP. Класс AnimationContainerDecoder с помощью встроенного декодера mgGif парсит байты анимаций и генерирует массив кадров Texture2D, которые затем добавляются в атлас как последовательные текстурные плашки.

Инверсия оси Y

⚠️ Внимание: Различие систем координат

Сервер: Ось Y направлена вниз. Координата Y = 0 соответствует самой верхней строке карты.
Unity: Стандартная декартова система координат. Ось Y направлена вверх, координата Y = 0 соответствует низу сцены.

Все пространственные преобразования координат между сетью и игровым миром централизованы в утилите CoordinateUtils:

// Конвертация серверной координаты Y в координату Unity
public static float ServerToUnityY(int serverY, int worldHeight)
{
    return worldHeight - 1 - serverY;
}

// Конвертация координаты Unity в серверную координату Y
public static int UnityToServerY(float unityY, int worldHeight)
{
    return worldHeight - 1 - (int)System.Math.Floor(unityY);
}
